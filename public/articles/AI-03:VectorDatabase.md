# 向量数据库

**向量数据库**（Vector Database）是一种专门用于存储和管理高维向量数据的数据库系统。随着人工智能和机器学习技术的发展，向量数据库在处理大规模数据集、实现高效相似性搜索和推荐系统等方面发挥着越来越重要的作用。

## 向量从哪里来？

通常有 **Embedding Model**（嵌入模型）将文本、图像、音频等非结构化数据转换为向量表示。常见的嵌入模型包括：
- **文本嵌入**：如 OpenAI 的 text-embedding-3-small
- **图像嵌入**：如 CLIP 模型
- **音频嵌入**：如 Wav2Vec
- **多模态嵌入**：如 Flamingo 模型
这些模型将输入数据映射到高维向量空间，使得语义相似的数据在向量空间中距离较近。

## 向量数据库能做什么？

典型场景： 

1. **相似性搜索** ：   通过计算向量之间的距离（如欧氏距离、余弦相似度），快速找到与查询向量最相似的数据项。
   * “和这句话意思最接近的文档有哪些？”
2. **RAG（Retrieval-Augmented Generation）**：检索生成增强，结合向量数据库和生成模型，实现基于检索的文本生成，提高生成内容的相关性和准确性。
   - ChatGPT + 私有文档问答
3. 推荐系统：基于用户和物品的向量相似度，进行个性化推荐。
   - 相似商品 / 相似内容推荐
4. 相似图片 / 音频检索

5. 去重 / 聚类 / 异常检测

## 用 RAG 举例

### RAG 的核心思路

RAG 的核心是：

> **先检索相关文档，再生成回答**

传统 LLM（如 ChatGPT）只能依靠训练数据回答，如果知识库外的内容它“没见过”，就会胡编。而 RAG 可以：

1. 先把用户问题对应的语义向量去向量数据库里找相似文档
2. 把这些文档作为上下文提供给 LLM，再生成回答

**流程公式化**：

```
用户问题 → Embedding → 向量数据库搜索 → 获取相关文档 → LLM生成答案
```

### 完整架构示意

```
+-------------------+
| 用户问题输入      |
+-------------------+
          |
          v
+-------------------+
| Embedding 模型     |  ← 例如 OpenAI text-embedding-3-large
+-------------------+
          |
          v
+-------------------+
| 向量数据库        |  ← Milvus / Pinecone / Chroma
| (检索最相似文档)  |
+-------------------+
          |
          v
+-------------------+
| 文档拼接成上下文  |
+-------------------+
          |
          v
+-------------------+
| LLM 生成答案       |  ← GPT / LLaMA / MPT
+-------------------+
          |
          v
+-------------------+
| 返回给用户         |
+-------------------+
```
